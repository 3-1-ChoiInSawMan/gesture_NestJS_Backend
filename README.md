# 제스처(Jesture) 백엔드 서비스

제스처의 NestJS 백엔드입니다.

## 협업 흐름

```
이슈 생성 → 브랜치 생성 → PR 요청 → 코드 리뷰 → main 머지
```

1. **이슈 생성**: 작업 전 이슈 템플릿에 맞게 이슈 작성
2. **브랜치 생성**: 이슈 번호 기반으로 브랜치 생성 (`feat/#이슈번호-작업명`)
3. **PR 요청**: 작업 완료 후 PR 생성, 이슈 연결
4. **코드 리뷰**: 상호 코드 리뷰 후 승인
5. **머지**: 승인 완료 시 `main`에 머지

### 브랜치 네이밍

| 타입 | 형식 | 예시 |
|------|------|------|
| 기능 추가 | `feat/#이슈번호-작업명` | `feat/#12-auth-login` |
| 버그 수정 | `fix/#이슈번호-작업명` | `fix/#15-token-refresh` |
| 리팩토링 | `refactor/#이슈번호-작업명` | `refactor/#20-auth-module` |

### 커밋 메시지

```
타입: 작업 내용 요약

예시)
feat: 로그인 API 구현
fix: JWT 토큰 만료 처리 오류 수정
refactor: AuthModule 구조 개선
```
